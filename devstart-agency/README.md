# DevStart Agency — Dossier de Travail
## Bloc 2 · Module 1 · Académie de Programmation IFRI L1

---

## 🎯 Contexte

Vous venez d'être recruté comme consultants juniors par **DevStart Agency**.

Un stagiaire a développé la page d'accueil du site de l'agence avant de partir précipitamment.
Le code **fonctionne** (à peu près), mais personne dans l'équipe ne comprend vraiment ce qu'il a fait.
Pas de commentaires, des noms incompréhensibles, une organisation douteuse.

**Votre mission : analyser, comprendre et documenter ce code.**
Pas le corriger — juste le comprendre et l'expliquer clairement.

---

## 📁 Contenu de ce dossier

```
devstart-projet/
│
├── original/
│   ├── index.html       ← La page HTML du stagiaire
│   └── style.css        ← La feuille de style du stagiaire
│
└── README.md            ← Ce fichier
```

---

## 📋 Ce que vous devez produire

| Livrable | Détail |
|---|---|
| 📄 Rapport PDF | 5 à 8 pages — analyse complète du code |
| 💻 Dépôt GitHub | Code original + version annotée par vos soins |
| 🎤 Présentation orale | 10 min devant l'académie |

---

## 🗓️ Planning de la semaine

| Jour | Travail |
|---|---|
| Jour 1 | Prise en main, lecture du code, création du dépôt GitHub |
| Jour 2 | Annotations — expliquez chaque section avec vos mots |
| Jour 3 | Analyse critique — qu'est-ce qui pose problème ? |
| Jour 4 | Rédaction du rapport + préparation des slides |
| Jour 5 | Finalisation + présentation devant l'académie |

---

## ✅ Règles importantes

- **Ne modifiez pas le code original.** Copiez-le dans un dossier `/annotated` pour vos annotations.
- **Tout s'explique avec vos mots.** Pas de copier-coller depuis internet.
- **GitHub est obligatoire.** Commits réguliers avec des messages clairs.
- **Tout le monde intervient** lors de la présentation orale.

---

## 💡 Par où commencer ?

1. Ouvrez `index.html` dans votre navigateur — regardez le résultat visuel.
2. Ouvrez ensuite `index.html` dans un éditeur de code (VS Code).
3. Essayez de faire le lien entre ce que vous voyez à l'écran et le code.
4. Posez-vous la question : *"Qu'est-ce que cette partie fait ? Pourquoi ?"*

Bonne analyse ! 🚀
